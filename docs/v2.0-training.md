# V2.0 Training

V2.0 training is split by enterprise role.

## Operators

- Deploy with the production configuration runbook.
- Verify `/health`, `/ready`, `/version`, and metrics after deployment.
- Run release certification, backup/restore checks, and support bundle
  generation before handoff.
- Escalate security incidents through the incident runbook.

## Administrators

- Create users and service accounts with least-privilege roles.
- Grant project access by role; do not share admin accounts.
- Review audit logs for authentication failures, permission denials, support
  access, exports, deletes, and service-account activity.
- Verify retention and delete policies before onboarding a tenant.

## Researchers And Reviewers

- Use project-scoped dashboards and exports only for authorized projects.
- Treat generated molecules as computational hypotheses.
- Treat model predictions, docking outputs, graph paths, portfolio outputs,
  benchmarks, and prospective validation analytics as prioritization or
  evaluation artifacts, not biomedical evidence.
- Submit operational feedback separately from scientific evidence.

## Completion Criteria

Training is complete when each role can run its assigned workflow in the
synthetic demo environment, explain the scientific-integrity boundaries, and
identify where to find the relevant runbook.
