# V2.0 Validation Evidence Packages

V2.0 validation evidence packages collect deterministic software and process
evidence for enterprise release certification.

## Required Contents

- Release manifest with version, commit, contract identifiers, limitations, and
  gate status.
- Release check report from `molecule-ranker release check`.
- Deterministic release validation report from `molecule-ranker validate release`.
- Golden workflow reports produced with mocked services and synthetic data.
- Security and guardrail audit summaries.
- API and artifact contract exports.
- Deployment readiness, health, and version endpoint evidence.
- Backup/restore verification evidence.
- Support bundle manifest with redacted diagnostics only.
- Operator runbook references and release certification notes.

## Prohibited Contents

Evidence packages must not contain plaintext secrets, bearer tokens, API keys,
service credentials, full imported assay files, copyrighted article text, fake
citations, fake assay results, generated biomedical truth, synthesis
instructions, lab protocols, dosage guidance, or patient treatment guidance.

## Interpretation

These packages validate software behavior, release process, deployment
readiness, controls, and operations. They do not clinically validate the
platform and do not prove molecule activity, safety, efficacy, binding,
synthesizability, or treatment suitability.
