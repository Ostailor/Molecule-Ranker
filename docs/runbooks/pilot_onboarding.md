# Pilot Onboarding Runbook

Use this runbook to onboard internal pilot users without changing scientific
scope.

1. Confirm the deployment reports version `2.0.0` and V1 contract identifiers.
2. Confirm users understand that molecule-ranker is for internal research use
   only.
3. Confirm generated molecules, rankings, hypotheses, model predictions,
   docking outputs, benchmark outputs, and Codex summaries remain review and
   planning artifacts.
4. Create pilot users with least-privilege roles.
5. Review dashboard, job, audit, support, and feedback workflows with the pilot
   owner.
6. Record pilot feedback in the pilot feedback workflow, not as evidence or
   assay results.

Do not request or collect API keys, service tokens, environment variables,
cache files, credentials, or secrets in onboarding notes.
