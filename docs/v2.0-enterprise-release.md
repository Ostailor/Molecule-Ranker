# V2.0 Enterprise Release

V2.0 ships molecule-ranker as a validated enterprise discovery operating
system. The release hardens the V1.9 pilot-ready platform for enterprise
operations; it does not add major new science modules.

## Release Contract

- Package version: `2.0.0`.
- Active release stage: `validated_enterprise_discovery_operating_system`.
- Stable API contract: `api.v1`.
- Stable artifact contract: `artifacts.v1`.
- Stable integration data-contract family: `data-contracts.v1`.
- Stable warehouse schema: `mr_warehouse_v1.0.0`.

The V1 API and artifact identifiers remain stable in V2.0 so enterprise clients
can upgrade the platform release without adopting a breaking API generation.
Additive response fields and optional request fields remain permitted under the
existing contract compatibility notes.

## Certification Gates

Release certification must cover:

- Stable enterprise release contracts.
- Production-grade deployment.
- Security hardening.
- Enterprise identity and access controls.
- Tenant and project isolation.
- Validation evidence packages.
- Operational runbooks.
- Disaster recovery and backup verification.
- Governance and audit readiness.
- Performance and reliability targets.
- Release certification workflows.
- Stable APIs and SDK behavior.
- Enterprise admin controls.
- Full end-to-end synthetic demo workflows.
- V2.0 documentation and training material.

Run the certification checks with:

```bash
uv run molecule-ranker release check --json
uv run molecule-ranker validate release --json
uv run molecule-ranker release manifest --output release_manifest.json
uv run molecule-ranker release notes --output RELEASE_NOTES.md
```

## Non-Goals

V2.0 does not expand generation, docking, ADMET, graph reasoning, model
training, integrations, or campaign planning except for stability, validation,
security, and enterprise readiness.

V2.0 does not provide medical advice, patient treatment guidance, dosage
guidance, synthesis instructions, lab protocols, fake evidence, fake assay
results, fake citations, Codex-generated biomedical truth, generated molecule
activity/safety/efficacy claims, docking overclaims, model prediction
overclaims, benchmark overclaims, or prospective validation overclaims.

Enterprise validation artifacts are software/process validation artifacts, not
clinical validation.
