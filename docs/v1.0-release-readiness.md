# V1.0 Release Readiness

V1.0 ships molecule-ranker as a validated internal research platform MVP. It is
a stabilization, validation, release-readiness, and operational-quality release.
It is not a feature-expansion release.

## Scope

V1.0 hardens the V0.9 platform surface:

- Source-backed disease, target, molecule, and literature retrieval.
- Generated molecule hypotheses that remain computational hypotheses.
- Developability triage without expanded ADMET claims.
- Experimental feedback loop and expert review workflow.
- Codex CLI as a guarded LLM backbone, never as biomedical truth.
- Hosted mode with auth, RBAC, projects, jobs, secure worker, integrations,
  webhooks, audit logs, and deployment packaging.

V1.0 does not add major science capabilities, model families, generation
expansion, ADMET expansion, or new integration families.

## Release Gates

The canonical gate list is `molecule_ranker.release.V1_RELEASE_GATES`.

Required categories:

- End-to-end golden workflows.
- Release-quality validation suite.
- Security and guardrail verification.
- Data provenance and reproducibility checks.
- Integration contract tests.
- Hosted deployment hardening.
- Platform documentation.
- Admin/operator runbooks.
- Demo project and synthetic example artifacts.
- Versioned API and artifact contracts.
- Backup/restore and disaster recovery checks.
- V1.0 release packaging.

## Validation Command Set

Run before tagging V1.0:

```bash
uv sync --all-groups --frozen
uv run ruff check .
uv run pyright
uv run pytest
```

Targeted release evidence:

```bash
uv run pytest tests/test_v1_release_readiness.py
uv run pytest tests/test_server_hardening.py tests/test_deployment_packaging.py
uv run pytest tests/test_integrations_v09.py tests/test_integration_security.py
uv run pytest tests/test_webhook_ingestion.py
uv run pytest tests/test_codex_evals.py tests/test_security_regressions_v08.py
```

## Scientific Integrity

All V1.0 artifacts and demos must preserve these constraints:

- No medical advice, patient treatment guidance, dosage, synthesis
  instructions, or lab protocols.
- No fabricated evidence, assay results, or fake citations.
- No Codex-generated biomedical truth.
- No generated molecules presented as validated actives.

Generated molecules must remain labeled as computational hypotheses. Imported
experimental results must be user-supplied, linked to exact structures, and
quality-controlled before any score adjustment.

## Release Package

A V1.0 release package includes:

- Git tag and package version `1.0.0`.
- Wheel/source artifact created from the locked dependency graph.
- Container image digest and deployment manifests.
- Test, lint, typecheck, and release-readiness command output.
- OpenAPI output and V1 contract identifiers.
- Synthetic demo artifacts from `examples/v1.0-demo/`.
- Backup/restore dry-run record for the target deployment mode.
