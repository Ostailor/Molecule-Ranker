# User Guide

Users work inside permitted projects to view source-backed rankings, generated
hypotheses, developability triage, experiments, reviews, models, structure
workflows, portfolios, graphs, hypotheses, campaigns, evaluations, and project
artifacts.

## Standard Workflow

1. Select a project you are authorized to access.
2. Review source provenance and artifact metadata.
3. Treat generated molecules as labeled hypotheses.
4. Treat model and docking outputs as separate, bounded artifacts.
5. Route uncertain or generated outputs to review before export or downstream
   use.
6. Keep decisions separate from Codex summaries and evaluation outputs.

## User Responsibilities

- Do not upload patient treatment guidance, dosing content, synthesis routes,
  lab protocols, real secrets, or unauthorized data.
- Do not relabel generated molecules as active, safe, effective, binding, or
  synthesizable.
- Do not treat model predictions, docking scores, benchmark results, or Codex
  summaries as evidence.

## Support

Use the support process for access issues, job failures, artifact validation
errors, integration failures, or suspected redaction problems. Support bundles
must be redacted and should not include raw secrets or Codex transcripts unless
explicitly approved by policy.
