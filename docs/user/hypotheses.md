# Hypothesis Planning

molecule-ranker V1.6 hypothesis planning is for internal research use only. It
provides no medical advice, no clinical claims, no lab protocols, no synthesis
instructions, and no dosing. Generated molecules require validation.

The hypothesis engine turns graph-backed records into reviewable planning
artifacts. It can draft mechanistic, molecule-target, generated-molecule
follow-up, developability-risk, assay-contradiction, scaffold/series,
evidence-gap, active-learning, portfolio-decision, and high-level validation
question hypotheses.

A hypothesis is not evidence. A research question is not a lab protocol. A
validation plan is not an experimental procedure. Hypotheses carry uncertainty,
contradiction checks, evidence gaps, falsification criteria, ranking metadata,
lifecycle events, and review status so reviewers can decide what still needs
expert attention.

Codex may help draft hypothesis text from supplied graph context, but
deterministic validation must confirm every referenced entity, relation,
citation, result, and artifact. Codex outputs are assistant artifacts only and
must not create evidence, assay results, citations, graph nodes, graph edges,
mechanisms, scores, review decisions, or biomedical truth claims.
