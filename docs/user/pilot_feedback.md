# Pilot Feedback

Pilot feedback captures usability, reliability, performance, onboarding,
dashboard, support, deployment, and documentation issues.

Pilot feedback is not biomedical evidence. It is not an assay result, molecule,
score, benchmark result, review decision, or Codex-generated scientific claim.
Do not use feedback to claim that molecules are safe, active, effective,
synthesizable, binding, or clinically useful.

When filing feedback, include the workflow, page or command, request ID or job
ID when available, expected behavior, observed behavior, and whether the issue
blocked pilot work. Do not include secrets, environment variables, cache files,
credentials, API keys, service tokens, lab protocols, synthesis instructions,
dosing, or patient treatment guidance.
