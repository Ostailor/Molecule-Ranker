# V2.0 Enterprise Contracts

V2.0 is an enterprise release over the stable V1 API and artifact contracts.
The platform version is `2.0.0`; the API route family remains `/api/v1`.

## Stable Identifiers

- API contract version: `api.v1`.
- API route version: `v1`.
- Artifact contract version: `artifacts.v1`.
- Integration data-contract family: `data-contracts.v1`.
- Warehouse schema contract: `mr_warehouse_v1.0.0`.

These identifiers are intentionally unchanged in V2.0. The release validates
enterprise deployment and governance readiness without introducing a breaking
API generation.

## Compatibility Rules

- Required request fields, required response fields, route paths, auth
  requirements, permission requirements, and error envelope semantics are stable.
- Additive response fields and optional request fields are allowed.
- Breaking changes require a new contract identifier and migration guidance.
- SDK clients must surface server contract versions from `/version` and
  `/api/v1/version`.

## Evidence Boundary

Contract exports, OpenAPI exports, release manifests, support bundles, and
validation packages are software/process evidence. They are not clinical
validation, biomedical evidence, assay results, generated molecule activity
claims, safety claims, efficacy claims, docking proof, or model prediction
proof.
