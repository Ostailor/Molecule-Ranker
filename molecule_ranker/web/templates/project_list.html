First-run setup

First-run checklist

Create a project workspace before running jobs, registering artifacts, or inviting reviewers. Projects hold runs, artifacts, review decisions, Codex assistant outputs, and audit events.

  • Generation is disabled by default.
  • Docking is disabled by default.
  • External writes are disabled by default.
  • Codex is disabled until configured.
  • Codex worker jobs are scoped and permissioned.
  • Review workflow is optional.
  • Experimental evidence is disabled unless configured and imported.
  • Integrations default to dry-run/read-only.

Project setup wizard

What can I do next?

  • Create a project, then import or register source-backed artifacts.
  • Open Review to triage candidates and generated hypotheses separately.
  • Use Evaluation to inspect platform-quality artifacts, not research evidence.
  • Keep integrations in dry-run/read-only mode until an admin enables writes.
